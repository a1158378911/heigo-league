# 爬虫扩展指南

将论坛爬虫部署到你的 Windows 云服务器，实现自动抓取。

---

## 📋 前提条件

| 条件 | 说明 |
|------|------|
| Windows 云服务器 | 已有自己的服务器 |
| 论坛账号 | soccerbar.cc 账号密码 |
| 简道云配置 | 与网页使用同一个应用 |

---

## 🚀 部署步骤

### 第 1 步：上传爬虫代码到服务器

将 `fm26-league-crawler` 文件夹上传到服务器，例如：
```
D:\fm26-league-crawler\
```

### 第 2 步：安装 Python

1. 访问 https://www.python.org/downloads/
2. 下载 Python 3.10+
3. 安装时勾选 **"Add Python to PATH"**

验证安装：
```bash
python --version
```

### 第 3 步：安装依赖

```bash
cd D:\fm26-league-crawler
pip install -r requirements.txt
playwright install chromium
```

### 第 4 步：配置爬虫

编辑 `config.py`：

```python
# 论坛账号
FORUM_CONFIG = {
    'username': '你的论坛账号',
    'password': '你的论坛密码',
    'board_urls': [
        'https://www.soccerbar.cc/forum-100-1.html',
    ],
}

# 简道云配置（与网页一致）
JIANDAOYUN_CONFIG = {
    'app_id': '你的 App ID',
    'api_key': '你的 API Key',
    'table_matches': '比赛记录表 ID',
    'table_teams': '球队信息表 ID',
}

# 运行时间（每天 22:00）
CRAWLER_CONFIG = {
    'run_time': '22:00',
}
```

### 第 5 步：测试运行

```bash
python crawler.py
```

查看输出，确认抓取和上传正常。

### 第 6 步：设置开机自启

#### 方式 A：使用任务计划程序（推荐）

1. 搜索「任务计划程序」并打开
2. 右侧点击「创建基本任务」
3. 名称：`FM26 联赛爬虫`
4. 触发器：选择「每天」
5. 时间：`22:00`
6. 操作：选择「启动程序」
7. 程序/脚本：`python.exe` 的路径
8. 添加参数：`crawler.py`
9. 起始于：`D:\fm26-league-crawler`
10. 完成

#### 方式 B：使用调度器

```bash
python scheduler.py
```

保持窗口运行，或使用 NSSM 注册为 Windows 服务。

## ⚙️ 配置说明

### 爬虫数据 vs 网页数据

| 特性 | 爬虫数据 | 网页录入 |
|------|----------|----------|
| 数据来源 | 论坛抓取 | 手动录入 |
| 状态 | 待审核 | 已确认 |
| 标记 | "论坛爬虫" | "人工录入" |
| 去重 | 自动检测 | 手动避免 |

### 去重逻辑

爬虫会检查以下条件判断是否重复：
- 主队相同
- 客队相同
- 比分相同

满足以上条件则跳过，避免重复录入。

## 🔧 常见问题

### Q: 爬虫登录失败？

A:
1. 检查账号密码是否正确
2. 论坛可能需要验证码
3. 删除 `cookies.json` 重新登录

### Q: 抓取不到数据？

A:
1. 检查板块 URL 是否正确
2. 查看 `crawler.log` 日志
3. 论坛结构可能变化

### Q: 爬虫和网页数据冲突？

A:
- 不会冲突，都写入同一个表
- 通过「数据来源」字段区分
- 积分榜同时统计两者

### Q: 如何修改抓取频率？

A:
编辑 `config.py`：
```python
CRAWLER_CONFIG = {
    'run_time': '22:00',  # 改为其他时间
}
```

或在 `scheduler.py` 中添加多个时间：
```python
schedule.every().day.at("10:00").do(run_crawler)
schedule.every().day.at("22:00").do(run_crawler)
```

## 📊 监控爬虫

### 查看日志

```bash
type crawler.log
```

或直接用记事本打开 `crawler.log`。

### 检查数据

1. 登录简道云
2. 进入「比赛记录表」
3. 筛选「数据来源」= "论坛爬虫"
4. 查看最新抓取的数据

### 告警设置（可选）

在简道云设置「智能助手」：
- 触发条件：爬虫数据连续 3 次失败
- 执行动作：发送邮件/短信通知

## 🎯 最佳实践

### 1. 数据审核

建议流程：
```
爬虫抓取 → 待审核 → 管理员确认 → 已确认 → 计入积分
```

在简道云设置审核流程，确保数据准确。

### 2. 备份策略

- 每天爬虫前自动备份简道云数据
- 使用简道云的「数据导出」功能
- 或调用 API 定期导出

### 3. 错误处理

爬虫失败时的应对：
1. 检查日志定位问题
2. 手动录入比赛结果（用网页）
3. 修复爬虫后重新运行

### 4. 性能优化

- 抓取频率不要太高（建议每天 1-2 次）
- 设置合理超时时间
- 使用 Cookie 保持登录态

---

## 📞 技术支持

遇到问题：
1. 查看 `crawler.log` 日志文件
2. 检查简道云 API 配置
3. 确认论坛账号正常

如需调整解析逻辑，修改 `crawler.py` 中的 `parse_match_content()` 函数。

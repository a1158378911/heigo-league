# FM26 联赛管理系统 - 部署指南

通过免费云平台，让网页可以通过网址访问。

---

## 🎯 推荐方案：Vercel

**网址示例：** `https://fm26-league.vercel.app`

### 优势

- ✅ 完全免费
- ✅ 国内访问快
- ✅ 自动 HTTPS
- ✅ 无需服务器
- ✅ 5 分钟完成

---

## 📦 方案一：Vercel 拖拽部署（最简单）

### 步骤

#### 1. 访问 Vercel

打开 https://vercel.com

#### 2. 注册账号

- 点击「Sign Up」
- 可用 GitHub/邮箱注册

#### 3. 安装 Vercel CLI（可选）

```bash
npm install -g vercel
```

#### 4. 部署

**方式 A：使用 CLI**

```bash
# 进入项目目录
cd fm26-league-web

# 部署
vercel
```

**方式 B：网页拖拽**

1. 登录 Vercel
2. 点击「Add New...」→「Project」
3. 把整个 `fm26-league-web` 文件夹拖到上传区域
4. 等待部署完成

#### 5. 获取网址

部署完成后会显示：
```
https://fm26-league-web-xxx.vercel.app
```

可以自定义项目名：
1. 进入项目设置
2. 点击「Domains」
3. 编辑网址

## 📦 方案二：Netlify 部署

### 步骤

#### 1. 访问 Netlify

打开 https://netlify.com

#### 2. 注册账号

- 点击「Sign Up」
- 可用 GitHub/邮箱注册

#### 3. 部署

**方式 A：拖拽部署**

1. 登录后进入「Sites」
2. 把 `fm26-league-web` 文件夹拖到上传区域
3. 等待部署完成

**方式 B：连接 GitHub**

1. 把代码上传到 GitHub
2. 在 Netlify 选择该仓库
3. 点击「Deploy site」

#### 4. 自定义网址

1. 进入「Site settings」
2. 点击「Change site name」
3. 输入自定义名称

网址格式：`https://自定义名称.netlify.app`

---

## 📦 方案三：GitHub Pages

### 步骤

#### 1. 创建 GitHub 仓库

1. 登录 https://github.com
2. 创建新仓库：`fm26-league-web`
3. 设为公开（Public）

#### 2. 上传文件

```bash
# 进入项目目录
cd fm26-league-web

# 初始化 git
git init

# 添加文件
git add .

# 提交
git commit -m "Initial commit"

# 关联远程仓库
git remote add origin https://github.com/你的用户名/fm26-league-web.git

# 推送
git push -u origin main
```

#### 3. 启用 GitHub Pages

1. 进入仓库 → Settings → Pages
2. Source 选择「main branch」
3. 点击「Save」

#### 4. 获取网址

等待几分钟后，网址会显示：
```
https://你的用户名.github.io/fm26-league-web
```

## 🌐 自定义域名（可选）

### Vercel 自定义域名

1. 进入项目设置 → Domains
2. 输入你的域名
3. 按提示配置 DNS

### Netlify 自定义域名

1. 进入 Site settings → Domain management
2. 点击「Add custom domain」
3. 按提示配置 DNS

### DNS 配置示例

```
类型    名称    值
A       @       76.76.21.21
CNAME   www     cname.vercel-dns.com
```

## 🔒 安全说明

### 数据存储在简道云

- 网页只负责展示和录入
- 所有数据存在你的简道云账号
- 部署平台不存储任何数据

### 配置保存在浏览器

- API 配置保存在用户浏览器 localStorage
- 不同设备需要重新配置
- 不会泄露给第三方

### HTTPS 加密

- Vercel/Netlify 自动提供 HTTPS
- 数据传输加密
- 安全可靠

## 🆘 常见问题

### Q: 部署后无法访问？

A: 
1. 检查文件是否完整上传
2. 确保 `index.html` 在根目录
3. 查看部署日志是否有错误

### Q: 国内访问慢？

A:
- Vercel 在国内速度较快
- 如果慢可以试试 Netlify
- 或考虑部署到国内 CDN

### Q: 可以多人同时使用吗？

A:
- 可以，无人数限制
- 数据都在简道云，实时同步
- 每人需要在自己浏览器配置 API

### Q: 网址太长能改吗？

A:
- Vercel/Netlify 都支持自定义项目名
- 或绑定自己的域名

### Q: 部署后如何更新？

A:

**CLI 部署：**
```bash
vercel --prod
```

**GitHub 部署：**
```bash
git push
```

**拖拽部署：**
重新拖拽文件夹覆盖
